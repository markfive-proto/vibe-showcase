Understanding Props

Props (short for "properties") are the primary mechanism for passing data from a parent component to a child component in React. They are read-only and help you create reusable, composable components.

Think of props like function arguments -- they let you customize how a component behaves and what it renders.

function Welcome(props) {
return <h1>Hello, {props.name}</h1>;
}
// Usage:
<Welcome name="Sara" />

Key Rules of Props

  • Props are passed from parent to child (one-way data flow)
  • Props are read-only -- never modify props directly
  • Any JavaScript value can be passed as a prop
  • Use destructuring for cleaner syntax
